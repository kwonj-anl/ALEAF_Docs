# GTEP Expansion Outputs

## Purpose
This page documents the files produced by the expansion export workflow in A-LEAF.

The source entry point is:
- [export_LC_GTEP_result_EXP](C:/ALEAF/a-leaf/src/model/LCO_GTEP.jl)

That function:
1. optionally writes a JSON snapshot of the solved expansion model
2. calls `report_LCO_GTEP_result_EXP!`
3. writes a family of CSV reports for expansion, dispatch, scarcity, power flow, representative days, and summary outputs

## Expansion Export Workflow
`export_LC_GTEP_result_EXP` produces:
- optional JSON output
- CSV reports generated by:
  - `report_result_expansion_gen_GTEP`
  - `report_result_expansion_line_GTEP`
  - `report_result_scarcity_ens_GTEP`
  - `report_result_scarcity_reserve_GTEP`
  - `report_result_power_flow_expansion_GTEP`
  - `report_result_dispatch_expansion_GTEP`
  - `report_result_tech_summary_expansion_GTEP`
  - `report_result_system_summary_expansion_GTEP`
  - `report_result_repdays_GTEP`

## File List

### Optional JSON snapshot
- `ALEAF_LC_GTEP_EXP_<test_system_name>_<Case_ID>_<timestamp>.json`

Written only when:
- `Simulation Setting["export_model_reference_json_expansion_flag"] == true`

Contents:
- `expansion model result`
- `expansion model system reference`
- `network data`
- `setting`

This is the largest and most detailed export. It is useful for reproducibility and post-run debugging, but it is not the main reporting format.

### CSV files always produced by the expansion reporting workflow

- `<Case_ID>__expansion_line_result.csv`

- `<Case_ID>__scarcity_reserve_result_EXP.csv`
- `<Case_ID>__power_flow_EXP.csv`
- `<Case_ID>__dispatch_EXP_of_year_<stage>.csv`
- `<Case_ID>__market_EXP.csv`
- `<Case_ID>__policy_EXP.csv`
- `<Case_ID>__demand_response_EXP.csv`
- `<Case_ID>__system_tech_summary_planning_stage_EXP.csv`

## Common Field Conventions

### `Scenario`
The case name written into the model reference.

### `year`
The planning-stage id, not necessarily the literal calendar year.

### `Start Year`
The actual calendar year at which a planning stage begins.

### `Number of Years`
The stage length represented by that planning stage.

### `day`
The representative-day id used in the reduced chronology.

### `hour`
The hour index used inside the reduced representative-day dispatch.

### `time`
The intra-hour or time-slice index. Many studies use a single time slice, but the export format always keeps this dimension.

### `Stochastic_scenario_ID`
The stochastic expansion scenario id when stochastic expansion is enabled. Otherwise it is reported as `0`.

### `NumDays`
The representative-day weight. It is the number of full-year days represented by that reduced day.

## 1. Generator Expansion Output

### File
- `<Case_ID>__expansion_gen_result.csv`

### Columns
- `Scenario`: case identifier.
- `unit_id`: internal generator index from `gen_index`.
- `year`: planning-stage id.
- `u_new_G_iy`: number of new generating units added in that stage.
- `u_ret_G_iy`: number of units retired in that stage.
- `u_G_iy`: total number of units in service in that stage after additions and retirements.
- `u_new_ESH_iy`: storage-duration expansion decision for storage technologies.
- `u_ESE_iy`: total storage energy-duration quantity carried by the unit in that stage.

## 2. Transmission Expansion Output

### File
- `<Case_ID>__expansion_line_result.csv`

### Columns
- `Scenario`: case identifier.
- `line_id`: internal branch index.
- `year`: planning-stage id.
- `f_bus`: from-bus index used in the model.
- `t_bus`: to-bus index used in the model.
- `f_bus_name`: exported bus label for the from side.
- `t_bus_name`: exported bus label for the to side.
- `merged_line_UIDs`: original line identifiers merged into the modeled branch, if present.
- `original_rate`: original transmission rating before expansion, in MW after per-unit conversion.
- `u_new_T_ky`: transmission expansion decision for that branch and stage.
- `final_rate`: post-expansion transmission rating. The code computes this as `(1 + u_new_T_ky) * original_rate`.
- `length`: branch length used in transmission investment cost accounting.

## 3. Scarcity ENS Output

### File
- `<Case_ID>__scarcity_ens_result_EXP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `node`: bus index.
- `ens_ndhty`: energy not served at the bus, day, hour, time, and stage.

## 4. Scarcity Reserve Output

### File
- `<Case_ID>__scarcity_reserve_result_EXP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `zone`: reserve-zone id.
- `rns_spin_zdhty`: reserve shortage for contingency or spinning reserve in that reserve zone.
- `rns_nonspin_zdhty`: reserve shortage for non-spinning reserve.
- `rns_flex_up_zdhty`: upward flexible reserve shortage.
- `rns_flex_dn_zdhty`: downward flexible reserve shortage.

## 5. Power Flow Output

### File
- `<Case_ID>__power_flow_EXP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `line_id`: internal branch index.
- `f_bus`: from-bus index.
- `t_bus`: to-bus index.
- `f_region`: exported label for the from side. In practice this is the bus label used by the report writer.
- `t_region`: exported label for the to side. In practice this is the bus label used by the report writer.
- `orinal_rate`: original branch rating before expansion, after per-unit conversion. The column name is spelled this way in the code.
- `expansion`: transmission expansion decision `u_new_T_ky` used for that branch and stage.
- `final_rate`: expanded branch capacity after applying the expansion decision.
- `flow`: dispatched branch flow.
- `LMP_from_bus`: nodal price at the from bus.
- `LMP_to_bus`: nodal price at the to bus.
- `congestion_flag`: indicator showing whether the line is congested.
- `line_losses`: line-loss value reported by the model.
- `wheeling_cost`: congestion price difference converted into a wheeling-cost style output.
- `line_length`: branch length.
- `NumDays`: representative-day weight.

## 6. Dispatch Output

### File pattern
- `<Case_ID>__dispatch_EXP_of_year_<stage>.csv`

One dispatch CSV is written per planning stage.

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `Stochastic_scenario_ID`: stochastic expansion scenario id, or `0`.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `unit_id`: internal generator index.
- `PLANT_NAME`: generator name used for reporting.
- `bus_id`: modeled bus index.
- `Bus_Name`: bus label.
- `Parent_Bus_Name`: parent or higher-level bus label from the region configuration.
- `Region_Name`: configured region name for the bus.
- `Tech_ID`: technology id for the unit.
- `UnitGroup`: grouped technology label such as `wind_ons`, `pv`, `gas_cc`, or storage groups.
- `Unit_Category`: broad category such as thermal, renewable, or storage.
- `Unit_Type`: finer technology type used in the model data.
- `u_G_iy`: total unit count in service in that stage.
- `u_ESE_iy`: storage energy-duration quantity for storage technologies.
- `ICAP`: installed capacity in MW for the reporting row.
- `sto_c_idhty`: commitment-state style storage status variable when present.
- `c_idhty`: unit commitment on/off state when commitment is modeled.
- `su_idhty`: startup indicator when commitment is modeled.
- `g_idhty`: generation output in MW.
- `reg_up_idhty`: regulation-up provision.
- `reg_dn_idhty`: regulation-down provision.
- `spin_idhty`: spinning reserve provision.
- `nonspin_idhty`: non-spinning reserve provision.
- `flex_up_idhty`: upward flexible reserve provision.
- `flex_dn_idhty`: downward flexible reserve provision.
- `curt_idhty`: curtailment output.
- `chg_idhty`: storage charging power.
- `soc_idhty`: storage state of charge.
- `hybrid_chg_idhty`: charging or energy-transfer term for hybrid resources.
- `hybrid_type`: hybrid resource label used by the generator index.
- `Fuel_Type`: fuel label.
- `Fuel_Consumption_MMBtu`: implied fuel use from dispatched generation and heat rate.
- `Fuel_Cost`: fuel-cost component attributed to the row.
- `Inertia`: inertia contribution based on the unitâ€™s online state and configured inertia constant.
- `MC`: marginal-cost style value used for the generator row.
- `NumDays`: representative-day weight.

### File

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `PLANT_NAME`: demand response resource name.
- `bus_idx`: bus index.
- `bus_name`: bus label.
- `Region_Name`: configured region name.
- `Parent_Bus_Name`: parent or higher-level bus label.
- `Online_Year`: online year for the demand response resource.
- `UNITGROUP`: demand-response group label.
- `UNIT_CATEGORY`: category label.
- `UNIT_TYPE`: type label.
- `CAP`: MW capacity.
- `INTERCON_LIM`: interconnection limit.
- `Integer_Flag`: whether the demand-response resource is modeled with integer structure.
- `Daily_DR_Limit_MWh`: daily energy limit for the resource.
- `Num_DR_Segments`: number of demand-response segments.
- `Pct_MW_1` to `Pct_MW_5`: MW size of each demand-response segment.
- `Price_1` to `Price_5`: segment prices after economic-base conversion.
- `Hybrid_Gen`: hybrid generation linkage flag or identifier.
- `Hybrid_Gen_CAP`: linked hybrid generation capacity.
- `Hybrid_ES`: linked hybrid storage flag or identifier.
- `Hybrid_ES_CAP`: linked hybrid storage capacity.
- `lfl_lt`: baseline large flexible load level.
- `lfl_DR_lt`: dispatched demand-response amount.
- `lfl_seg_lt_1` to `lfl_seg_lt_5`: segment-level dispatched DR quantities.
- `lfl_ind_lt_1` to `lfl_ind_lt_5`: segment activation indicators.
- `lfl_g_G_LFL_lt`: local generation serving the flexible load.
- `lfl_g_G_Grid_lt`: grid-supplied energy serving the flexible load.
- `lfl_g_G_ES_lt`: storage discharge serving the flexible load.
- `lfl_g_ES_LFL_lt`: energy from the flexible load into storage or associated hybrid accounting term.
- `lfl_g_ES_Grid_lt`: grid-to-storage charging term for the flexible load resource.
- `lfl_chg_Grid_ES_lt`: explicit grid-to-storage charge quantity.
- `lfl_soc_lt`: flexible-load storage state of charge.

## 12. Representative-Day Selection Output

### File
- `<Case_ID>_repday_selection_result_EXP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `RepDay_id`: internal representative-day id used by the model.
- `Stochastic_scenario_ID`: stochastic scenario identifier for the representative day when stochastic expansion is enabled. Otherwise `0`.
- `NumDays`: number of full-year days represented by the representative day.
- `Daygroup_ID`: grouped-day id that the representative day belongs to.
- `Numdays_Daygroup`: total number of full-year days represented by the full day group.
- `Reference_Day`: original calendar day used as the representative day. This comes from the representative-day field `Day`.

In multi-round mode, these values are reported from:
- `multi_round_info["planning_stages"][year][day_id]`

In standard mode, they are reported from:
- `repdays`
