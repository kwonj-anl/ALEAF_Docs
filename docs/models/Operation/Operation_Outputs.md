# GTEP Operation Outputs

## Purpose
This page documents the files produced by the GTEP operation export workflow in A-LEAF.

The source entry point is:
- [export_LC_GTEP_result_OP](C:/ALEAF/a-leaf/src/model/LCO_GTEP.jl)

That function:
1. optionally writes a JSON snapshot of the solved operation run
2. calls `report_LCO_GTEP_result_OP!`
3. writes a family of CSV reports for dispatch, market, policy, demand response, power flow, summaries, and representative-day reporting

## Operation Export Workflow
`export_LC_GTEP_result_OP` produces:
- optional JSON output
- CSV reports generated by:
  - `report_result_dispatch_operation_GTEP`
  - `report_result_power_flow_operation_GTEP`
  - `report_result_tech_summary_operation_GTEP`
  - `report_result_system_summary_operation_GTEP`
  - `report_result_repdays_OP_GTEP`

## File List

### Optional JSON snapshot
- `ALEAF_LC_GTEP_OP_<test_system_name>_<Case_ID>_<timestamp>.json`

Written only when:
- `Simulation Setting["export_model_reference_json_operation_flag"] == true`

Contents:
- `operational model result`
- `operational model system reference`
- `network data`
- `setting`

### CSV files produced by the operation reporting workflow
- `<Case_ID>__dispatch_OP_of_year_<stage>.csv`
- `<Case_ID>__market_OP.csv`
- `<Case_ID>__policy_OP.csv`
- `<Case_ID>__demand_response_OP.csv`
- `<Case_ID>__power_flow_OP.csv`
- `<Case_ID>__system_tech_summary_OP.csv`
- `<Case_ID>__system_summary_OP.csv`
- `<Case_ID>__system_summary_planning_stage_OP.csv`
- `<Case_ID>__system_summary_annual_OP.csv`
- `<Case_ID>_repday_selection_result_OP.csv`

## Common Field Conventions

### `Scenario`
The case name written into the system reference.

### `year`
The planning-stage id.

### `day`
The representative-day id in the reduced chronology.

### `hour`
The hour index inside the representative day.

### `time`
The time-slice index inside the hour.

### `NumDays`
The representative-day weight used to scale reduced-day outputs back toward annual values.

## 1. Dispatch Output

### File pattern
- `<Case_ID>__dispatch_OP_of_year_<stage>.csv`

One dispatch CSV is written per planning stage.

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `unit_id`: internal generator index.
- `PLANT_NAME`: generator name used for reporting.
- `bus_id`: modeled bus index.
- `Bus_Name`: bus label.
- `Parent_Bus_Name`: parent or higher-level bus label.
- `Region_Name`: configured region name.
- `Tech_ID`: technology id.
- `UnitGroup`: grouped technology label.
- `Unit_Category`: technology category.
- `Unit_Type`: technology type.
- `u_G_iy`: total unit count in service in the stage.
- `u_ESE_iy`: storage energy-duration quantity for storage technologies.
- `ICAP`: installed capacity in MW for the unit row.
- `sto_c_idhty`: storage commitment-style status variable when present.
- `c_idhty`: unit commitment on/off state when commitment is modeled.
- `su_idhty`: startup indicator.
- `g_idhty`: generation output in MW.
- `reg_up_idhty`: regulation-up provision.
- `reg_dn_idhty`: regulation-down provision.
- `spin_idhty`: spinning reserve provision.
- `nonspin_idhty`: non-spinning reserve provision.
- `flex_up_idhty`: upward flexible reserve provision.
- `flex_dn_idhty`: downward flexible reserve provision.
- `curt_idhty`: curtailment output.
- `chg_idhty`: storage charging power.
- `soc_idhty`: storage state of charge.
- `hybrid_chg_idhty`: hybrid charging or transfer term.
- `hybrid_type`: hybrid resource type label.
- `Inertia`: inertia contribution from the resource.
- `MC`: marginal-cost style value for the row.
- `NumDays`: representative-day weight.

## 2. Market Output

### File
- `<Case_ID>__market_OP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `bus_id`: modeled bus index.
- `Bus_Name`: bus label.
- `Parent_Bus_Name`: parent or higher-level bus label.
- `Region_Name`: configured region name.
- `load`: represented load at the bus and hour.
- `scarcity_E`: energy scarcity quantity.
- `scarcity_SPIN`: spinning reserve scarcity quantity.
- `scarcity_NSPIN`: non-spinning reserve scarcity quantity.
- `scarcity_FU`: upward flexible reserve scarcity quantity.
- `scarcity_FD`: downward flexible reserve scarcity quantity.
- `LMP`: locational marginal price.
- `RCP_RU`: regulation-up reserve clearing price.
- `RCP_RD`: regulation-down reserve clearing price.
- `RCP_Spin`: spinning reserve clearing price.
- `RCP_NSpin`: non-spinning reserve clearing price.
- `RCP_FU`: upward flexible reserve clearing price.
- `RCP_FD`: downward flexible reserve clearing price.
- `NumDays`: representative-day weight.

## 3. Policy Output

### File
- `<Case_ID>__policy_OP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `bus_id`: modeled bus or policy-zone id used by the reporting loop.
- `slack_CEG_ndy`: clean-energy-generation-target slack quantity.

The operation policy file only reports the clean-energy slack column here; it does not include the RPS slack column used by the expansion policy report.

## 4. Demand Response Output

### File
- `<Case_ID>__demand_response_OP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `PLANT_NAME`: demand response resource name.
- `bus_idx`: bus index.
- `bus_name`: bus label.
- `Region_Name`: configured region name.
- `Parent_Bus_Name`: parent or higher-level bus label.
- `Online_Year`: online year of the demand response resource.
- `UNITGROUP`: demand-response group label.
- `UNIT_CATEGORY`: category label.
- `UNIT_TYPE`: type label.
- `CAP`: MW capacity.
- `INTERCON_LIM`: interconnection limit.
- `Integer_Flag`: whether the demand response resource is modeled with integer structure.
- `Daily_DR_Limit_MWh`: daily demand-response energy limit.
- `Num_DR_Segments`: number of demand-response segments.
- `Pct_MW_1` to `Pct_MW_5`: MW size of each demand-response segment.
- `Price_1` to `Price_5`: segment prices.
- `Hybrid_Gen`: linked hybrid generation flag or identifier.
- `Hybrid_Gen_CAP`: linked hybrid generation capacity.
- `Hybrid_ES`: linked hybrid storage flag or identifier.
- `Hybrid_ES_CAP`: linked hybrid storage capacity.
- `lfl_lt`: baseline flexible-load level.
- `lfl_DR_lt`: dispatched demand-response level.
- `lfl_seg_lt_1` to `lfl_seg_lt_5`: segment-level dispatched DR quantities.
- `lfl_ind_lt_1` to `lfl_ind_lt_5`: segment activation indicators.
- `lfl_g_G_LFL_lt`: local generation serving the flexible load.
- `lfl_g_G_Grid_lt`: grid-supplied energy serving the flexible load.
- `lfl_g_G_ES_lt`: storage discharge serving the flexible load.
- `lfl_g_ES_LFL_lt`: flexible-load-to-storage or linked hybrid accounting term.
- `lfl_g_ES_Grid_lt`: grid-to-storage charging term.
- `lfl_chg_Grid_ES_lt`: explicit grid-to-storage charge quantity.
- `lfl_soc_lt`: flexible-load storage state of charge.

## 5. Power Flow Output

### File
- `<Case_ID>__power_flow_OP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `day`: representative-day id.
- `hour`: representative-day hour.
- `time`: time-slice id.
- `line_id`: internal branch index.
- `f_bus`: from-bus index.
- `t_bus`: to-bus index.
- `f_region`: exported label for the from side.
- `t_region`: exported label for the to side.
- `orinal_rate`: original line rating before expansion. The column name is spelled this way in the code.
- `expansion`: transmission expansion quantity inherited from the expansion state.
- `final_rate`: line capacity after applying the expansion quantity.
- `flow`: dispatched power flow.
- `LMP_from_bus`: locational price at the from bus.
- `LMP_to_bus`: locational price at the to bus.
- `congestion_flag`: indicator that the line is congested.
- `line_losses`: reported line losses.
- `wheeling_cost`: congestion-price-difference style wheeling-cost output.
- `line_length`: branch length.
- `NumDays`: representative-day weight.

## 6. Technology Summary Output

### File
- `<Case_ID>__system_tech_summary_OP.csv`

### Columns
- `Scenario`: case identifier.
- `Stage`: planning-stage id.
- `Start Year`: actual start year for the planning stage.
- `Number of Years`: stage length.
- `PLANT_NAME`: plant name.
- `Bus_ID`: bus index.
- `Bus_Name`: bus label.
- `Parent_Bus_Name`: parent or higher-level bus label.
- `Region_Name`: configured region name.
- `Tech_ID`: technology id.
- `UnitGroup`: grouped technology label.
- `Unit_Category`: technology category.
- `Unit_Type`: technology type.
- `Fuel`: fuel label.
- `TotalUnits`: total units in service in the operation run.
- `NewUnits`: units added in the linked expansion result, if one exists.
- `RetUnits`: units retired in the linked expansion result, if one exists.
- `ICAP`: installed capacity.
- `UCAP`: unforced capacity after applying capacity credit.
- `ICap_New`: installed capacity associated with additions.
- `ICap_Ret`: installed capacity associated with retirements.
- `UCap_New`: unforced capacity associated with additions.
- `UCap_Ret`: unforced capacity associated with retirements.
- `Storage_MWh`: storage energy capacity.
- `Storage_Hr`: storage duration.
- `Generation`: annualized generation.
- `Curtail`: annualized curtailment.
- `Storage_Charge_MWh`: annualized charging energy.
- `Reserve_RegUp`: annualized regulation-up provision.
- `Reserve_RegDn`: annualized regulation-down provision.
- `Reserve_Spin`: annualized spinning reserve provision.
- `Reserve_NSpin`: annualized non-spinning reserve provision.
- `Reserve_FlexUp`: annualized upward flexible reserve provision.
- `Reserve_FlexDn`: annualized downward flexible reserve provision.
- `Generation_Cost`: generation operating cost.
- `Charge_Cost`: charging cost.
- `Regulation_Cost`: regulation reserve cost.
- `Spin_Cost`: spinning reserve cost.
- `Nspin_Cost`: non-spinning reserve cost.
- `Flex_Cost`: flexible reserve cost.
- `UnitRevenue_E`: energy-market revenue.
- `UnitRevenue_AS`: ancillary-service revenue.
- `UnitRevenue_CRED`: credit-related or policy-related revenue.
- `UnitRevenue`: total revenue.
- `UnitProfit`: net revenue minus reported operating costs.
- `FuelConsumption`: annualized fuel consumption.
- `FuelCost`: annualized fuel cost.
- `FOM`: fixed O&M cost.
- `CAPCRED`: capacity credit.
- `Reference_Annual_Gen_Investment_Cost`: reference annual generator investment basis.
- `Gen_Investment_Cost`: discounted generator investment cost from the linked expansion state.
- `Gen_ITC`: discounted investment tax credit from the linked expansion state.

## 8. Representative-Day Selection Output

### File
- `<Case_ID>_repday_selection_result_OP.csv`

### Columns
- `Scenario`: case identifier.
- `year`: planning-stage id.
- `RepDay_id`: internal representative-day id.
- `NumDays`: number of full-year days represented by that representative day.
- `Daygroup_ID`: grouped-day id that the representative day belongs to.
- `Numdays_Daygroup`: total number of full-year days represented by the full day group.
- `Reference_Day`: original calendar day mapped to that representative day.
